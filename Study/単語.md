- `繰り返し(くりかえし)` : 반복 / 반복문 전반을 지칭할 때 가장 기본어
- `制御フロー(せいぎょふろー)` : 제어 흐름 / break, continue가 바꾸는 “실행의 흐름”을 설명할 때 자주 사용
- `変数スコープ(へんすう すこーぷ)` : 변수 유효 범위 / 로컬 변수 접근 가능 범위(버그 포인트)
- `無限ループ(むげんるーぷ)` : 무한 루프 / `for(;;)` 또는 조건 갱신 실패로 발생하는 장애 유형

- `継続(けいぞく)` : 계속/지속 / continue 동작을 설명할 때 “ループを継続する”
- `スキップ` : 건너뛰기 / “この処理はスキップします” (continue의 실무 표현)
- `ガード節(がーどせつ)` : 가드 절 / 실패 케이스를 먼저 처리하고 핵심 로직을 깔끔하게 유지
- `可読性(かどくせい)` : 가독성 / continue 남용 vs 가드/메서드 분리 논의에서 핵심

- `連想配列 (れんそうはいれつ)` : 연관 배열. 일본 실무에서는 Map이나 Dictionary 구조를 흔히 이렇게 부릅니다.
- `重複 (ちょうふく)` : 중복. "Setはデータの重複を許しません(Set은 데이터의 중복을 허용하지 않습니다)" 처럼 자주 쓰입니다.
- `ハッシュ衝突 (はっしゅしょうとつ)` : 해시 충돌. 면접에서 HashMap의 원리를 설명할 때 반드시 등장하는 키워드입니다.

- `入力検証(にゅうりょくけんしょう)` : 입력 검증 / continue가 가장 빛나는 영역
- `スキップ理由(すきっぷりゆう)` : 스킵 사유 / 운영 로그·집계에서 자주 쓰는 표현
- `ラベル付きcontinue(らべるつきこんてぃにゅー)` : 라벨 continue / 레코드 단위 탈락 처리에 사용
- `保守性(ほしゅせい)` : 유지보수성 / continue 남용 시 떨어지는 품질 지표

- `集計(しゅうけい)` : 집계 / Map으로 카운팅할 때 자주 쓰는 말
- `デフォルト値(でふぉるとち)` : 기본값 / getOrDefault의 default
- `ラベル(らべる)` : 라벨 / ラベル付きcontinue・ラベル付きbreak

- `継承(けいしょう)` : 상속 / 클래스 관계를 말할 때 기본어
- `親クラス(おやくらす) / 子クラス(こくらす)` : 부모/자식 클래스
- `多態性(たたいせい)` : 다형성 / override 메서드가 런타임 객체 기준으로 호출되는 성질
- `フィールド隠蔽(ひでんいんぺい)` : 필드 숨김(field hiding) / 필드가 override되지 않는다는 점을 강조할 때

- `静的バインディング(せいてき)` : 정적 바인딩 / 컴파일 타임에 결정(필드, static 메서드)
- `動的ディスパッチ(どうてき)` : 동적 디스패치 / 런타임 객체 기준(오버라이딩 메서드)
- `隠蔽(いんぺい)` : 숨김(hiding) / 필드 숨김, static 메서드 숨김을 말할 때
- `多態性(たたいせい)` : 다형성 / “메서드에만 제대로 적용된다”를 강조

- `構文エラー (こうぶんえらー)` : 구문 에러 (Syntax Error). 오타나 문법 틀림 등으로 인해 컴파일이 되지 않는 에러.
- `論理エラー (ろんりえらー)` : 논리 에러 (Logical Error). 실행은 되지만 결과값이 개발자의 의도와 다르게 나오는 버그.
- `境界値テスト (きょうかいちてすと)` : 경계값 테스트. 오늘이 정확히 생일일 때처럼 조건문의 아슬아슬한 기준점을 테스트하는 기법.

- `タイポ (Typo)` : 오타. "変数名にタイポがあります (변수명에 오타가 있습니다)"처럼 매일 쓰는 단어입니다.
- `比較 (ひかく)` : 비교. 두 날짜나 문자열을 비교할 때 사용하는 단어입니다.
- `小なり / 大なり (しょうなり / だいなり)` : 작다(`<`) / 크다(`>`). 부등호 기호를 읽는 일본어 표현입니다.

- `ヌルポ (Nurupo)` : NullPointerException의 일본 IT 업계 은어(줄임말)입니다. 실무에서 "あ、ヌルポで落ちた(아, 널포로 죽었네)"라고 정말 매일 씁니다.
- `配列の範囲外 (はいれつのはんいがい)` : 배열의 범위 밖. `IndexOutOfBoundsException`을 설명할 때 쓰는 표현입니다.
- `キャスト例外 (きゃすとれいがい)` : 형변환(Cast) 예외. `ClassCastException`을 뜻합니다.

- `通信的凝集度 (つうしんてき ぎょうしゅうど)` : 통신적(통합/교환적) 응집도. 동일한 데이터를 다루는 모듈의 특성.
- `逐次的凝集度 (ちくじてき ぎょうしゅうど)` : 순차적 응집도. 파이프라인처럼 이어지는 특성. (`順次的`라고도 씁니다)
- `機能的凝集度 (きのうてき ぎょうしゅうど)` : 기능적 응집도. 단일 목적을 가진 최고 품질의 모듈.
